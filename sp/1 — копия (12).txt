
ЭКЗАМЕНАЦИОННЫЙ БИЛЕТ №13


Вопросы: Что такое и чем отличаются login shell, non-login shell, interactive и non-interactive shell? Для чего нужна команда su? Какие файлы задействованы в формировании окружения?
Login Shell (Логин-Оболочка):
Что это: Запускается при входе пользователя в систему.
Пример: Вход через терминал или SSH.

Non-Login Shell (Не логин-оболочка):
Что это: Запускается без входа пользователя.
Пример: Открытие нового терминала в графической среде.

Interactive Shell (Интерактивная оболочка):
Что это: Взаимодействие с пользователем в терминале.
Пример: Ввод команд в терминале.

Non-Interactive Shell (Неинтерактивная оболочка):
Что это: Выполнение команд из сценария без взаимодействия.
Пример: Запуск скрипта.

Команда su:
Что это: Переключение пользователя в текущем терминале.
Пример: su username для переключения на пользователя username.

Файлы окружения:
Login Shell: .bash_profile, .bash_login, .profile.
Non-Login Shell (Interactive): .bashrc, .bash_aliases.
Non-Interactive Shell (Script): Окружение зависит от содержания самого скрипта.

Практическое задание:
Создайте LVM, состоящий из двух разделов и одного целого диска. Внутри группы томов создайте два логических раздела, каждый из которых должен занимать 30% пространства группы томов.
sudo pvcreate /dev/sda1 /dev/sdb1 физический том
sudo vgcreate myvg /dev/sda1 /dev/sdb1 группа томов
sudo lvcreate -n mylv1 -l 30%VG myvg логические тома
sudo lvcreate -n mylv2 -l 30%VG myvg
sudo mkfs.ext4 /dev/myvg/mylv1 форматирование логических томов
sudo mkfs.ext4 /dev/myvg/mylv2
